
## 二、技术架构（合规双链设计）

### 2.1 双链架构概览

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              应用层 Application Layer                              │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐                      │
│   │ GPU插件 │    │存储插件 │    │代理插件 │    │其它插件 │                      │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                          插件运行时 Plugin Runtime                                  │
│          插件管理       │       资源抽象       │       计量计费                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│                            P2P 网络层 P2P Network                                  │
│      ┌──────────────────────┐           ┌──────────────────────┐                  │
│      │     控制网络          │           │     数据网络          │                  │
│      │  节点发现/协商/心跳   │           │  隧道传输/负载均衡    │                  │
│      └──────────────────────┘           └──────────────────────┘                  │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   ┌─────────────────────────────────┐    ┌─────────────────────────────────┐     │
│   │         业务链 Business Chain      │    │      结算层 Settlement Layer     │     │
│   │         (联盟链/私有链)             │    │      (e-CNY 数字人民币)           │     │
│   ├─────────────────────────────────┤    ├─────────────────────────────────┤     │
│   │ • 节点注册与身份认证               │    │ • 资金托管 (Escrow)              │     │
│   │ • 资源注册与能力声明               │    │ • e-CNY 支付与收款                │     │
│   │ • 任务分发与匹配                   │    │ • 清结算与对账                    │     │
│   │ • 服务交付存证                     │    │ • 合规审计日志                    │     │
│   │ • 争议仲裁记录                     │    │ • 税务申报接口                    │     │
│   └─────────────────────────────────┘    └─────────────────────────────────┘     │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 双链职责分离

| 链层 | 技术选型 | 核心职责 | 合规设计 |
|------|---------|---------|----------|
| **业务链** | Hyperledger Fabric / Substrate | 任务调度、资源匹配、链上存证 | 联盟链，节点需实名认证 |
| **结算层** | e-CNY 2.0 (银行存款型) | 资金托管、支付清结算 | 通过持牌银行/机构接入 |

### 2.3 合规要点

```
⚖️ 合规设计 Compliance Design
│
├── 🚫 不发币原则
│   ├── 不发行 Token / 虚拟货币
│   ├── 不进行代币融资 (ICO/IEO/IDO)
│   └── 不提供虚拟货币兑换服务
│
├── ✅ e-CNY 合规接入
│   ├── 仅支持 e-CNY 数字人民币
│   ├── 通过银行/持牌机构接入支付
│   ├── 支持银行存款型 e-CNY (e-CNY 2.0)
│   └── 符合央行《数字人民币管理服务体系》2026版
│
├── 🔐 数据合规
│   ├── KYC/AML 实名认证
│   ├── 大额交易报告
│   └── 数据本地化存储
│
└── 📋 运营合规
    ├── ICP 许可证 (如需 web 服务)
    ├── 区块链信息服务备案
    └── 网络安全等级保护
```

### 2.4 分层职责

| 层级 | 组件 | 职责 |
|------|------|------|
| 应用层 | GPU/存储/代理插件 | 特定资源类型实现 |
| 运行时层 | Plugin Runtime | 生命周期管理、资源抽象 |
| 网络层 | Control/Data Plane | 节点发现、数据传输 |
| **业务链层** | Fabric/Substrate | 任务调度、存证、仲裁 |
| **结算层** | e-CNY 2.0 | 资金托管、支付清结算 |

```
├── 🦌 DeerFlow 集成
```
