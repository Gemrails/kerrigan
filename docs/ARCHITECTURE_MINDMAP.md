# Kerrigan v2 架构脑图

## 🦌 Kerrigan
> 可议价的去中心化任务分发网络

---

## 一、愿景与定位

```
🌟 愿景
├── 去中心化资源交易基础设施
├── 闲置计算资源共享 (CPU/GPU/存储/带宽)
├── 智能合约实现透明定价与结算
└── 数字人民币(e-CNY)支付

💰 核心价值
├── 去中心化 - 无单一控制点
├── 可议价 - 市场供需动态调节
├── 可信 - 链上合约保障
└── 可扩展 - 插件化架构
```

---

## 二、技术架构

### 2.1 四层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层 Application                          │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    │
│  │ GPU插件 │    │存储插件 │    │代理插件 │    │其它插件 │    │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘    │
├─────────────────────────────────────────────────────────────────┤
│                       插件运行时 Plugin Runtime                      │
│     插件管理      │      资源抽象      │      计量计费            │
├─────────────────────────────────────────────────────────────────┤
│                        P2P 网络层                                  │
│    ┌──────────────────┐        ┌──────────────────┐              │
│    │     控制网络       │        │     数据网络     │              │
│    │  节点发现/协商    │        │  隧道传输/加密   │              │
│    └──────────────────┘        └──────────────────┘              │
├─────────────────────────────────────────────────────────────────┤
│                       区块链层 Blockchain                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │ 资源注册合约  │  │ 交易结算合约  │  │ 插件仓库合约  │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
│                  e-CNY 数字人民币支付集成                          │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 分层职责

| 层级 | 组件 | 职责 |
|------|------|------|
| 应用层 | GPU/存储/代理插件 | 特定资源类型实现 |
| 运行时层 | Plugin Runtime | 生命周期管理、资源抽象 |
| 网络层 | Control/Data Plane | 节点发现、数据传输 |
| 区块链层 | Smart Contracts | 资源注册、交易结算、支付 |

## 三、P2P 网络架构

```
📡 P2P Network
│
├── 🔗 双层网络设计
│   │
│   ├── 🎛️ 控制平面 Control Plane
│   │   ├── 节点发现 (Kademlia DHT)
│   │   ├── 协议协商
│   │   ├── 心跳保活
│   │   └── 服务注册
│   │   └── 端口: 38888
│   │
│   └── 📊 数据平面 Data Plane
│       ├── 隧道传输
│       ├── 负载均衡
│       ├── 流量加密
│       └── 端口: 38889
│
├── 👤 节点角色
│   ├── Consumer - 资源消费者
│   ├── Provider - 资源提供者
│   ├── Relay - 流量中继
│   └── Validator - 交易验证者
│
└── 🧭 NAT 穿透
    ├── 直连 (公网IP)
    ├── 打洞 (hole punching)
    └── 中继 Fallback
```
